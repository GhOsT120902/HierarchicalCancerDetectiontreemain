 Getting Started

          
MedAI Dashboard runs medical scan images through a multi-stage AI pipeline and produces a structured clinical decision with visual explanations. Here is the typical workflow:

          
    
            
  1. Log in and land on the Dashboard tab.
    2. 
            
  2. Upload a scan image via drag-and-drop or the file picker.
    3. 
            
  3. Optionally enable the Manual Organ Override to force a specific organ context.
    4. 
            
  4. Click Run Intelligent Pipeline and wait for the pipeline to finish.
    5. 
            
  5. Review the four stage cards, the Grad-CAM heatmap, and the Final Decision banner.
    6. 
            
  6. Export a PDF report if needed.
    7. 
          

        


        


          
          

            
 Uploading a Scan

            

              Drag an image file onto the upload zone or click Select Image to open a file picker. A thumbnail preview appears immediately. Supported formats are JPEG, PNG, BMP, and TIFF. The file is sent to the backend only when you click Run Intelligent Pipeline — nothing is stored permanently.
            

          


          
          

            
 Manual Organ Override

            

              Toggle Manual Organ Override to bypass the AI's automatic organ selection and choose a specific organ from the dropdown. Use this when you already know the scan's anatomical context and want Level 2 / Level 3 analysis to be constrained accordingly. The Final Decision banner will note when an override was applied.
            

          


          
          

            
 Exporting a PDF Report

            

              After a successful analysis the Export Report button becomes active. Clicking it sends the current result to the backend, which assembles a structured PDF and saves it to your server's Documents folder. A confirmation message appears once the file is ready.
            

          


          
          

            
 Model Accuracy Tab

            

              The Model Accuracy tab runs every test image in the backend's test set through the full inference pipeline and computes per-stage accuracy metrics. Click Run Evaluation to start — the process can take several minutes on CPU. Results show accuracy cards and per-class breakdowns for each stage.
            

          


        


        
        

          
 Pipeline Stages

          


            

              
0

              

                Step 0 — Modality Validation
                
The first gate checks whether the uploaded image is a valid medical organ scan. If the image is a photograph, screenshot, or otherwise not an organ scan, the pipeline stops here and returns a Rejected Input decision. Only images that pass this check proceed to Level 1.

              

            


            

              
1

              

                Level 1 — Tissue / Organ Routing
                
The organ classification model identifies which organ or tissue type appears in the scan (e.g. lung, brain, kidney). The top prediction, along with a probability bar chart, is shown in the Level 1: Tissue card. A manual override replaces this prediction for all downstream stages.

              

            


            

              
2

              

                Level 2 — Normality Assessment
                
Given the identified organ, a normality model decides whether the tissue appears Normal or Abnormal. The confidence score and colour-coded status (green = Normal, red = Abnormal) are displayed in the Level 2: Normality card.

              

            


            

              
3

              

                Level 3 — Subtype Classification
                
For abnormal scans, the subtype model narrows the finding to a specific condition or pathology class (e.g. adenocarcinoma, glioma). The Level 3: Subtype card shows the predicted class, confidence, and a probability bar chart across all subtype classes.

              

            


          

        


        
        

          
 Grad-CAM Visualisation

          

            Gradient-weighted Class Activation Mapping (Grad-CAM) produces a heatmap overlay that highlights which regions of the image most influenced the model's decision. Warmer colours (red / orange) indicate high activation — the areas the model paid most attention to. This is shown alongside the original scan so you can visually verify whether the model focused on clinically relevant anatomy. The heatmap is generated for the final classification stage that produced the overall decision.
          

        


        
        

          
 Frequently Asked Questions

          


            

              What image formats are supported?
              
JPEG (.jpg / .jpeg), PNG (.png), BMP (.bmp), and TIFF (.tif / .tiff). Other formats may work if the browser can render them, but are not officially supported.

            


            

              What happens if the model is uncertain?
              
Low-confidence predictions are flagged in the warnings section below the stage cards. The Final Decision banner will still display the best prediction, but the confidence score and any warnings should be taken into account before acting on the result.

            


            

              Is my scan data stored or sent to a third party?
              
No. Images are processed entirely on the local server and are not persisted after the response is returned. No data is sent to external services.

            


            

              Why did my image fail the modality check?
              
The Step 0 gate is trained to detect valid organ scans. Natural photographs, diagrams, screenshots, and heavily compressed or watermarked images are likely to be rejected. Ensure your file is a genuine medical imaging scan.

            


            

              When should I use the manual organ override?
              
Use it when you have prior knowledge of the scan's anatomical source and the Level 1 prediction is incorrect or ambiguous. The override forces Levels 2 and 3 to use the selected organ's specialised models, which can improve accuracy in edge cases.

            


            

              Where is the exported PDF saved?
              
Reports are saved to the Documents folder on the machine running the backend server. The status message beneath the Export Report button confirms the file path once the report has been generated.
